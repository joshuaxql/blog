# 为什么要oh my openagent?
我用过claude，也用过gpt，还用过国产的deepseek、glm、minimax等等顶级的大模型，它们或多或少都有自己的优点和缺点。
那有什么办法可以扬长避短呢？oh my openagent给出了答案，把不同的任务分给擅长的模型，小任务由便宜好用的模型完成，复杂的任务
就用顶级的模型处理。

# 安装opencode
要使用oh my openagent，就要下载[opencode](https://opencode.ai/)，我推荐通过npm安装，确保电脑上有[nodejs](https://nodejs.org/zh-cn)环境。
```powershell
npm i -g opencode-ai
```
通过运行
```powershell
opencode
```
验证是否安装成功。
然后输入`/connect`来连接你购买的模型提供商。

# 安装oh my openagent
在opencode里输入
```
按照以下说明安装和配置 oh-my-opencode：
https://raw.githubusercontent.com/code-yeongyu/oh-my-opencode/refs/heads/master/docs/guide/installation.md
```
让opencode自动帮你下载安装。

# 配置oh my openagent
oh my openagent里有很多子智能体，下面是每个智能体的具体介绍：

| 类别                | 智能体                | 角色定位   | 推荐模型                              | 职责描述                                                         |
| :------------------ | :-------------------- | :--------- | :------------------------------------ | :--------------------------------------------------------------- |
| **核心智囊 (Core)** | **Sisyphus**          | 主编排器   | claude-opus-4-7 / kimi-k2.6 / glm-5.1 | 团队总指挥，负责拆解任务、分配工作、管理进度，确保任务彻底完成。 |
|                     | **Hephaestus**        | 深度工作者 | gpt-5.5                               | 正牌工匠，擅长端到端独立执行跨文件的复杂开发任务，代码质量高。   |
|                     | **Oracle**            | 架构顾问   | claude-opus-4-7 / kimi-k2.6 / glm-5.1 | 只读的AI架构师，用于代码审查、复杂调试和架构决策。               |
|                     | **Librarian**         | 研究员     | minimax-m2.7 / deepseek-v4-flash      | 图书管理员，负责多仓库分析、文档查询和查找开源实现示例。         |
|                     | **Explore**           | 代码搜索员 | minimax-m2.7 / deepseek-v4-flash      | 快速探索和搜索代码库，进行上下文感知的代码查找。                 |
|                     | **Multimodal-Looker** | 视觉专家   | gemini-3.1-pro                        | 专门分析图像、PDF、图表等多模态内容以提取信息。                  |

# 使用oh my openagent
- 在输入提示词前加上魔法词ulw，sisyphus主智能体就会尽可能调用子智能体。
- /ralph-loop指令可以让opencode对一个复杂的任务执行更多的循环
